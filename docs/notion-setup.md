# Notion CRM — настройка

Каждая заполненная анкета с `/test.html` создаёт новую страницу-карточку в Notion database. Database можно смотреть как kanban-доску (статусы — колонки), как таблицу или как gallery.

## Архитектура

```
quiz submit → /api/submit (Vercel) → POST https://api.notion.com/v1/pages → новая карточка
```

Что нужно от тебя:
1. Notion integration (создаёшь у себя)
2. Database в Notion с правильными properties
3. 2 env vars на Vercel

После этого каждая анкета автоматически появляется в Notion со статусом «Новая».

---

## Шаг 1. Создать Notion integration

1. Открой https://www.notion.so/profile/integrations
2. Жми **«+ New integration»**.
3. Заполни:
   - **Name:** `Алхимия — анкеты` (имя только для тебя, можно любое)
   - **Workspace:** твой workspace
   - **Type:** Internal
4. Жми **Submit**.
5. На странице созданной integration скопируй **«Internal Integration Secret»** (начинается с `secret_...` или `ntn_...`). Это твой `NOTION_TOKEN`. **Сохрани его** — он понадобится на шаге 5.

---

## Шаг 2. Создать database

1. В любой странице Notion создай новую database:
   - Введи `/database` → выбери **«Database — Full page»** (или «Inline»).
2. Назови её, например: `Анкеты Алхимии`.
3. Настрой properties. Имена и типы **должны точно совпадать** — иначе анкеты не сохранятся.

| Property | Тип | Опции / комментарий |
|---|---|---|
| **Имя** | Title | (автоматически первое поле) |
| **Статус** | Select | См. список опций ниже ⬇️ |
| **Email** | Email | |
| **Telegram** | URL | |
| **Архетип** | Select | См. список ⬇️ |
| **Профессия** | Select | См. список ⬇️ |
| **Доход** | Select | См. список ⬇️ |
| **Личная жизнь** | Select | См. список ⬇️ (раньше было «Отношения») |
| **Дети** | Select | См. список ⬇️ |
| **Сложности в работе** | Multi-select | Q2 из квиза ⬇️ |
| **Сложности в отношениях** | Multi-select | Q6 из квиза ⬇️ |
| **Мечта** | Text | (Rich text) |
| **Препятствия** | Text | |
| **Что пробовала** | Text | |
| **Токен** | Text | служебное поле, скрыть в view |

> ⚠️ **Schema change 2026-05-22.** Если ты создавала базу до этой даты — у тебя есть старые свойства **«Этап»** и **«Боли»**. В новой версии они **не используются**. Можно их удалить или оставить пустыми. И **обязательно создать новые свойства**: «Сложности в работе» (multi-select), «Сложности в отношениях» (multi-select). Старое «Отношения» переименовать в «Личная жизнь» — иначе новые анкеты не запишут эти поля.

> 💡 Создавать пустые select-опции заранее не обязательно. Notion создаёт опции автоматически при первом значении. Но если хочешь сразу задать **цвета** колонкам — добавь опции вручную (Шаг 2.5).

### Шаг 2.5 (опционально). Заранее настроить опции «Статус» с цветами

Кликни на property **Статус** в шапке → **«Edit property»** → добавь options:

| Статус | Цвет |
|---|---|
| Новая | yellow |
| Связались | blue |
| Назначено собеседование | purple |
| Собеседование прошло | pink |
| Приглашена | green |
| В программе | green (тёмный) |
| Отказала | gray |
| Не подходит | gray |
| На паузе | orange |

Опции для других select-property — Notion создаст автоматически, можешь потом раскрасить.

## Шаг 3. Дать integration доступ к database

1. Открой свою database.
2. Сверху справа жми `•••` → **«Connections»** → **«Connect to»**.
3. Найди в списке твою integration (`Алхимия — анкеты`) → выбери.
4. Notion может спросить подтверждение — соглашайся.

Без этого шага API не сможет писать в database. Самая частая ошибка.

---

## Шаг 4. Достать Database ID

1. Открой database в Notion.
2. Скопируй URL из адресной строки. Выглядит так:
   ```
   https://www.notion.so/yourworkspace/1a2b3c4d5e6f7890abcdef1234567890?v=...
                                       └──────────────┬───────────────┘
                                                Database ID
   ```
3. Database ID — это **32-символьная hex-строка** между последним `/` и `?`. Можно с дефисами `1a2b3c4d-5e6f-7890-abcd-ef1234567890` или без — Notion API принимает оба варианта.

---

## Шаг 5. Env vars на Vercel

Vercel → проект `alchemy` → **Settings → Environment Variables** → добавь два:

| Name | Value | Scope |
|---|---|---|
| `NOTION_TOKEN` | Integration secret из шага 1 | Production + Preview |
| `NOTION_DATABASE_ID` | Database ID из шага 4 | Production + Preview |

`NOTION_TOKEN` обязательно отметь как **Sensitive**.

После сохранения — **Redeploy** последний коммит (Vercel не редеплоит автоматом при изменении env).

---

## Шаг 6. Тест

1. Подожди ~1 мин пока пройдёт redeploy.
2. Открой https://wmnalchemy.com/test.html и пройди квиз тестово (имя «ТЕСТ-N», валидный email).
3. Сразу после сабмита открой Notion database. Должна появиться новая страница со статусом **«Новая»** и заполненными полями.

### Если не появилось

- Vercel → проект → **Logs** → последние минуты → ищи `Notion append failed`. Часто причины:
  - **401 unauthorized** → `NOTION_TOKEN` неверный или integration не подключена к database (шаг 3 пропущен).
  - **404 object_not_found** → `NOTION_DATABASE_ID` не тот, или integration не имеет доступа.
  - **400 validation_error** → property name не совпадает (например, ты назвала колонку «status» вместо «Статус»).
- Имена property **case-sensitive**, проверь по таблице из шага 2.

## Kanban view

После того как database заработает, делаем kanban:

1. В database сверху слева жми **«+ Add a view»** или иконку текущего view.
2. **Layout: Board.**
3. **Group by:** Статус.
4. Хочешь чтобы был только этот view по умолчанию — переименуй его в «Канбан» и перетащи на первую позицию.

Каждая карточка в kanban показывает превью полей. Чтобы выбрать какие поля видны на карточке — `•••` view → **«Card preview»** и **«Card properties»**.

## Что в карточке для удобства

Когда открываешь карточку (двойной клик), внутри:
- Все поля анкеты как property
- Контакты (Email и Telegram-ссылка кликабельны)
- Архетип, профессия, этап, доход
- Открытые ответы: мечта, препятствия, что пробовала

Прямо внутри карточки можно:
- Поменять статус (kanban-колонку)
- Добавить комментарии (правая панель)
- @-упомянуть Ольгу или Лизу
- Прикрепить файлы (записи звонков, договоры)
- Поставить задачу с дедлайном

## Дополнительные views (опционально)

В одной database можно создать несколько views — переключаются вверху.

**View 1 «Канбан»** — board по статусу. Главный рабочий.
**View 2 «Очередь»** — table, filter «Статус = Новая», sort по дате создания. Список того, что ждёт связи.
**View 3 «Мои»** — table, filter «Куратор = я». Для каждого ведущего отдельно.
**View 4 «Все»** — table без фильтров. Полная база.

## Безопасность

- **NOTION_TOKEN даёт write-доступ** к каждой database, с которой connected integration. Если token попадёт в публичный доступ — кто угодно сможет писать в эту database. Поэтому он Sensitive в Vercel и не виден в логах.
- **Если token утёк** — открой integration на https://notion.so/profile/integrations → **Reveal secret** → **Re-generate secret**. Старый сразу инвалидируется. Подмени в Vercel env, redeploy.

## Параллельно: TG, Sheets, Redis

Notion интеграция не заменяет другие пути. Сейчас при submit:
1. Анкета сохраняется в Redis на 24ч (для генерации LLM-письма)
2. Уведомление улетает в TG-чат Ольге
3. Если настроена Sheet → строка в Google Sheet
4. Если настроен Notion → карточка в Notion

Можно использовать одновременно. Notion для CRM, TG для real-time оповещений, Sheet для архива/отчётов.

Если Sheets не нужен — просто не задавай `GSHEETS_WEBHOOK_URL` в env, тот путь молча пропустится.
